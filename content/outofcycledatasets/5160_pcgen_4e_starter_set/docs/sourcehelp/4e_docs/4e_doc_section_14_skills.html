PCGen 4E Data Set Creation: Skills

We will cover the basics of skills as it relates to such systems as 4e(tm) and Starwars Saga(tm).

Here is an example of two skills

Monkey Vine Swinging

KEYSTAT:DEX

ACHECK:YES

TYPE:Dexterity

USEUNTRAINED:YES

BONUS:SKILLRANK|Monkey Vine Swinging|TL/2

VISIBLE:ALWAYS|READONLY

Monkey Lore

KEYSTAT:INT

ACHECK:NO

TYPE:Intelligence.Knowledge

USEUNTRAINED:YES

BONUS:SKILLRANK|Monkey Lore|TL/2

VISIBLE:ALWAYS|READONLY

Okay, so lets break these down.

KEYSTAT:<stat abbreviation> indicates whether or not the skill gets the bonus or penalty associated with the "stat" identified. If this is absent, the skill will not have any bonus or penalty from the stat.

ACHECK:<boolean> indicates whether there is an "Armor Check Penalty" (ACP) associated with this skill. Valid values are "YES" and "NO" (default). When you code your skills, any skill based upon strength, dexterity or constitution will use ACHECK:YES.

TYPE indicates the Skill type. So when viewing on the GUI it will display the skill under these types. In some systems, you would put the class name so you know which class this skill belongs to. CHOOSE:SKILLSNAMED|TYPE.<skill type> will pull up the associated Skills using the type included.

USEUNTRAINED:<boolean> indicates whether you may use the skill without purchasing ranks. Valid values are "YES" and "NO" (default). If the skill can not be used untrained, i.e. the value is "NO" or if the tag is not inclluded, the skill will not appear in the character's list of skills until one rank is purchased.

BONUS:SKILLRANK|<skill name>|TL/2; indicates that the skill named will gain a rank every two levels.

VISIBLE:ALWAYS|READONLY indicates that this skill will always be visible, but cannot be altered, i.e. you cannot add or remove skill ranks.

That covers a basic overview of the skills.

You need a skill for every skill you want in your game.
