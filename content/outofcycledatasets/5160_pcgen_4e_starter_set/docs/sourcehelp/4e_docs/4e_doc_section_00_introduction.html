How to Use This Guide

Common Conventions

Their are Common conventions that will hold true throughout this guide. Examples will be given as they should appear in the LST code. Any changes in Tag syntax will be maintained in the Main Program Docs. This guide is written for the Tag Syntax in use for 5.15.2.

Any time you see this '<>' two arrows back to back indicates a tab. For official PCGen lst files, tab spacing is 6. As you are making your set for personal use you may use any spacing you like. However, if you are asked to send your files to a helpful monkey, following the standards would make it easier for them.

Syntax should be followed exactly. YES is YES, NO is NO. Using No or YEs instead can cause the program not to register.

BONUS:VAR|SomeDude|0 is an example of a variable. If you write it out Bonus:var|someDude|0 it will not work. Capitalization and spelling are very exact in all cases. If you use DEFINE:SomeDUDE|0 and then use BONUS:VAR|SomeDude|1 Nothing will happen because your spelling and capitalization DO NOT Match. Also, NO SPACES are allowed at the BEGINNING or ENDING or any Tag or Syntax.

Commas should not be used EXCEPT inside Outputname. Feat/Ability/Equipment, etc. Do not like commas.
